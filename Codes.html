Codes

Codes exist for each InventoryItem, each RoomObject, and each Animation. They are mainly used as a means means of identifying objects, inventory and animation. This is because they are used as the first parameter of many methods of the BaseAction class.

If you use the CodeGenerator these are set to specially calculated ids. See SpecialPropertiesOfCodesMadeByCodeGeneration.

If not are crafting a list of ids by hand then you need to know the requirements of each codes:

  • The codes for ecah InventoryItem and RoomObject must be unique in the scope of a single room.
  • The codes for each Animation need also be unique in the scope of a single room.

Both the above points are because most of the methods of BaseAction use the codes to uniquely identify either an or a or an . For example: 

        case USE+ i.BOWL * o.FIRE_DEMON: // place bowl
                        return this
                        .walkTo(o.HARRY, fireDemonAfter)
                        .setActiveAnimation(a.HARRY_WALK_BACK)
                        .sleep(700)
                        .setInventoryVisible(i.BOWL,false)
                        .setVisible(o.BOWL_UNDER_FIRE, true)

In the above example, the first parameter of:

  • BaseAction.walkTo - is an object code.
  • BaseAction.setActiveAnimation - is an animation code.
  • BaseAction.setInventoryVisible - is an inventory code.
