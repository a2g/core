00122         @Override // method in animation
00123         public void onComplete() {
00124 
00125                 // It might seem like it's a better idea to 
00126                 // run this actions action completion handler 
00127                 // before calling startTheNextAction.
00128                 // However doDialogBranchAction starts execution of another animation tree
00129                 // and if order is 1) onCompleteGameAction 2) startTheNextAction,
00130                 // then on startTheNextAction starts triggering animations in the 
00131                 // newly replenished series of animations that onCompleteGameAction created,
00132                 // giving effectively two simultaneous thrads of
00133                 // executions of the Action chain. This results in an observed bug
00134                 // where only every second Action runs.
00135                 // So either a second action runner instance is needed for dialogs. Or, simpler 
00136                 // , the call order is kept like this.
00137                 this.callbacks.startTheNextAction(
00138                                 this);
00139 
00140                 onCompleteGameAction();
00141         }
